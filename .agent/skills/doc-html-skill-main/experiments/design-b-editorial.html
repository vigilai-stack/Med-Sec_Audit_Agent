The short version

Robot Framework is the Swiss Army knife of automation. rf-mcp hands that knife to your AI agent without anyone getting hurt. It is a Model Context Protocol server that turns a plain-language test idea into a real, reviewable Robot Framework suite — by discovering keywords, executing them interactively against a live execution context, checking the results, and only then writing the final .robot file.

Web, mobile, API, database: same workflow, different libraries.

It does not guess. Every keyword runs live before a single line is written down.

How it fits together

Deployment context

Runs as a local MCP server right next to your editor, or in Docker (ghcr.io/manykarim/rf-mcp), exposing MCP on port 8000 and an optional Django dashboard for watching sessions in real time.

№ 03

Key decisions

  1. Run keywords in a live, in-process RF context

    Keeps execution stateful and interactive — live variables, real page source, immediate feedback — instead of writing code blind and hoping.

    Adopted · ADR-020

  2. Offer three execution backends, not one

    In-process for speed and state, attach-to-live for reusing a debugger session, CLI for full and dry suite runs.

    Adopted

  3. Discover keywords by reflection, not runtime Libdoc

    Read live library instances directly with Python introspection — faster, and it sees exactly what is loaded.

    Adopted

  4. Optimize for small LLMs

    Tool profiles, intent-based actions, and response/state compression keep token budgets honest.

    Adopted

№ 05

Constraints

  • MCP STDIO shares one JSON-RPC channel, so Robot Framework console output is redirected and suppressed to keep the stream clean.
  • Requires Robot Framework 7.0+ — it leans on the RF 7.x running APIs.
  • The attach bridge listens on 127.0.0.1:7317 and expects an X-MCP-Token header.
  • Browser, mobile, and database features need the matching install extra.
