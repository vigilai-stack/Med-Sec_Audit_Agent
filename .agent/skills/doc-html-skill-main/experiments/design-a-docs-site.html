#The short version

Robot Framework is the Swiss Army knife of automation. rf-mcp hands that knife to your AI agent without anyone getting hurt. It is a Model Context Protocol server that turns a plain-language test idea into a real, reviewable Robot Framework suite — by discovering keywords, executing them interactively against a live execution context, checking the results, and only then writing the final .robot file.

ℹ️

Why this mattersGenerated tests are validated against a live RF context before they are written down — no blind code generation, no brittle output.

#Key decisions

Run keywords in a live, in-process RF context

adopted

Keeps execution stateful and interactive — live variables, real page source, immediate feedback — instead of writing code blind and hoping.

ADR-020

Offer three execution backends, not one

adopted

In-process for speed and state, attach-to-live for reusing a debugger session, CLI for full and dry suite runs.

Discover keywords by reflection, not runtime Libdoc

adopted

Read live library instances directly with Python introspection — faster, and it sees exactly what is loaded.

Optimize for small LLMs

adopted

Tool profiles, intent-based actions, and response/state compression keep token budgets honest.

#From plain words to a passing suite

  1. Describe the test in plain words

    The engineer states intent; the agent calls analyze_scenario and recommend_libraries.

  2. Discover the right keywords

    find_keywords returns matching keywords and their signatures from live library instances.

  3. Build it step by step

    Each execute_step runs in the live RF context and returns result, page source, and state.

  4. Write & run the suite

    build_test_suite emits a clean .robot file; run_test_suite reports pass or fail, with the receipts.

#Constraints

⚠️

STDIO is a shared channelMCP STDIO shares one JSON-RPC channel, so Robot Framework console output is redirected and suppressed to keep the stream clean.

⚠️

RF 7.0+ requiredThe server leans on the RF 7.x running APIs. The attach bridge listens on 127.0.0.1:7317 and expects an X-MCP-Token header.
